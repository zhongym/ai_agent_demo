---

# 目标
当用户询问销售额、订单量、客单价等经营指标时，先确认指标口径和时间范围，再调用 `data_center_query_metric`。

# 适用问题
- 我这个月销售额多少
- 上个月订单量多少
- 近30天客单价是多少

# 执行要求
1. 把用户问题映射到最接近的指标编码：
   - 销售额、营收、GMV 优先映射为 `sales_amount`
   - 订单量映射为 `order_count`
   - 客单价映射为 `unit_price`
2. 把时间表达转换成 `data_center_query_metric` 所需字段：
   - `time_range_label` 使用用户原始口径，例如 `本月`、`上个月`、`近30天`、`今日`
   - 同时补出 `start_date` 和 `end_date`，格式为 `YYYY-MM-DD`
   - 如果用户使用相对时间口径，例如 `本月`、`上个月`、`近30天`、`今日`，必须先调用 `get_time`，不要凭模型记忆猜当前日期
3. 如果用户没有明确时间范围，默认按 `本月`
4. 如果用户没有明确指标，先澄清，不要调用工具

# 输出要求
- 先给结论，再给统计区间
- 明确这是 mock 数据
- 提到更新时间
- 不要编造工具未返回的字段

# 输出要求
- 明确指标口径
- 明确时间范围
- 说明当前是 mock 数据
